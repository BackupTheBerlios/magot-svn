Features :

Functional :

  • Double-entry book-keeping, respecting standard equations :
    • Sum of Debit = Sum of Credit
    • Assets + Expenses = Liabilities + Incomes + Equity
    • Net Assets = Assets - Liabilities = Equity + Profits & Losses
  • Unconstrained Hierarchy of Accounts that compute their own balance as the sum of nested account balances.
  • Drag&Drop Accounts in the hierarchy.
  • An Account Ledger contains Entries. Each Entry defines a flow of data from/to its account.
  • A Transaction links several Entries and defines a single deal between several accounts.
  • Posting a modification of an Account/Entry/Transaction recompute all impacted balances.
  • Jump between Accounts of a Transaction (via their Entries).
  • Sort Entries by clicking on any column headers.
  • Multi-views of hierarchies & ledgers updated in real-time.
  • View accounts grouped by Dimension members under a new hierarchy : examples.
  • Automatic Profits & Losses and Net Assets balance computation.

          Technical :

  • Define a Business Domain Model, implemented with the PEAK framework.
  • Use WxPython as the cross-platform GUI toolkit. Tested on Linux and Windows.
  • Use Agile technics like YAGNI, DRY, Test-Driven and Domain-Driven Development, Object-Oriented Programming and Python of course!
  • Object Persistence is done with Python Pickling. Saves are transactionable with EditableFile.
  • Command Line Interface.
  • Lazy computation of any account balances.
  • MOLAP inspired technology.
  • MVC design pattern (several synchronized views of a single model).

Documentation :

The Business Model can be found here.

After the installation of Magot, type in a console magot newdb or magot newdb dimension to create a new database and magot gui to launch the user interface.

All Magot commands are shown below:

       newdb -- creates a new default database
    gui -- launchs the user interface
    accounts -- displays all accounts with their balances
    account <accountName> -- displays all entries of the account whose name is <accountName>
    check -- checks the accounting equation for all accounts
    addTx <desc debitAccount creditAccount amount [date, num]> -- adds a new Transaction

Example to list all entries of the account named 'Capital', just type in a console :

> magot account Capital

Installation :

Magot needs Python 2.4 and depends on WxPython (>=2.6.3.0) and PEAK (>=5.0a4).

1) download ez_setup.py

2) type in a console : python ez_setup.py -f http://peak.telecommunity.com/snapshots/ "PEAK>=0.5a4dev"

Magot is built with setuptools and can be installed with easy_install : 

3) type in a console : easy_install Magot or easy_install --upgrade Magot

You can now use Magot after reading the commands from the documentation. 

Type magot newdb dimension to create a new db with some dimensions defined in order to test the group by reporting.
