News :

Magot 0.1 has been released. It allows to :
  • Drag&Drop Accounts in the hierarchy.
  • Modify all fields of an Account, a Transaction or an Entry.
  • Change the opposite Account of an Entry.
  • Jump between Accounts following each Entry of a Transaction.
  • Sort Entries of an Account by clicking on any column headers.
For details, look at the history file.

Features :

Functional :

  • Double-entry book-keeping, respecting the equation : assets + expenses = liabilities + equity + revenue.
  • Hierarchy of drag-and-droppable Accounts that compute their own balance as the sum of nested balances.
  • Each Account holds Transactions.
  • Each Transactions holds Entries that link several Accounts for this Transaction.

Technical :

  • Define a Business Domain Model, implemented with the framework PEAK.
  • Use wxPython as the cross-platform GUI toolkit. Tested on Linux and Windows.
  • Use Agile technics like YAGNI, DRY, Test-Driven and Domain-Driven development, OOP and Python of course!
  • Object Persistence is done with Python Pickling.
  • Saves are transactionable with EditableFile.
  • Command lines.

Documentation :

The Business Model can be found here.

After the installation of Magot, type in a console magot newdb to create a new database and magot gui to launch the user interface.
All Magot commands are shown below:

    newdb -- creates a new default database
    gui -- launchs user interface
    accounts -- displays all accounts with their balances
    account <accountName> -- displays all entries of the account
    check -- checks the accounting equation
    addTx <desc debitAccount creditAccount amount [date, num]> -- adds a new Transaction

Example to list all entries of the account named 'Capital', just type in a console :
> magot account Capital

Installation :

Magot needs Python 2.4  and depends on WxPython 2.6.0.1 and PEAK (>=5.0a3).

Magot is built with setuptools and can be installed with easy_install by typing in a console : easy_install Magot-0.1-py2.4.egg.

You can now use Magot after reading the documentation.
